# Otium ビジョン・目的

## なぜ作る？

Otiumは、人々が日々の生活の中で感じたこと、体験したことを簡単に記録し、振り返ることができるツールとして開発されました。

現代社会では、日々の忙しさの中で、小さな幸せや気づきを見逃してしまいがちです。写真を撮ることはできても、その時の感情や思い出を言葉として残すことは難しく、時間が経つと忘れてしまうことがあります。

Otiumは、写真とテキストを組み合わせることで、より豊かな記録を残し、後から振り返ることで、自分の人生の軌跡を再発見できるようにすることを目指しています。

## 誰のため？

### ターゲットユーザー

1. **日記を書きたいが続かない人**
   - 文章だけでは記録が面倒に感じる人
   - 写真と組み合わせることで、より簡単に記録したい人

2. **自分の体験を振り返りたい人**
   - 過去の記録を見返して、自分の成長を実感したい人
   - 思い出を整理して、人生の軌跡を残したい人

3. **カテゴリー別に記録を整理したい人**
   - 仕事、プライベート、趣味など、テーマごとに記録を分類したい人
   - 特定のテーマの記録だけを見返したい人

4. **シンプルな記録ツールを求める人**
   - 複雑な機能は不要で、写真とテキストで記録できれば十分な人
   - 直感的に使えるシンプルなUIを好む人

## 何を解決？

### 解決する課題

1. **記録のハードルの高さ**
   - 従来の日記アプリは文章入力が中心で、記録するのが面倒
   - Otiumは写真を中心に、簡単なテキストで記録できるため、記録のハードルを下げる

2. **記録の散在**
   - 写真はスマホのギャラリー、メモは別アプリ、と記録が分散している
   - Otiumは写真とテキストを一つの記録として管理できる

3. **記録の整理の難しさ**
   - 大量の記録の中から特定の記録を見つけるのが困難
   - カテゴリー機能により、テーマごとに記録を整理・検索できる

4. **振り返りの機会の不足**
   - 記録を残しても、見返す機会がない
   - ギャラリー形式の表示により、視覚的に記録を振り返りやすくする

5. **プライバシーの懸念**
   - 個人の記録をクラウドサービスに預けることへの不安
   - JWT認証によるセキュアな認証と、ユーザーごとのデータ分離により、プライバシーを保護

### 提供する価値

1. **簡単な記録**
   - 写真を選んで、簡単なテキストを入力するだけで記録が作成できる
   - 日付は必須だが、タイトルや説明は任意で、記録のハードルを下げる

2. **視覚的な振り返り**
   - ギャラリー形式で記録を一覧表示
   - 写真を見ることで、その時の記憶を呼び起こしやすい

3. **カテゴリーによる整理**
   - 記録をカテゴリーで分類し、テーマごとに整理できる
   - カテゴリーフィルター機能により、特定のテーマの記録だけを表示可能

4. **プライバシーの保護**
   - 各ユーザーのデータは完全に分離され、他のユーザーからは見えない
   - JWT認証により、セキュアな認証を実現

5. **多様な設定**
   - 多言語対応（日本語・英語）
   - ライト/ダークモード対応
   - ユーザーが使いやすい環境を提供

## 将来のビジョン

### 短期目標

- 基本的な記録機能の安定化
- ユーザーフィードバックの収集と改善
- パフォーマンスの最適化

### 中期目標

- 記録の検索機能の強化
- 統計・インサイト機能の充実
- エクスポート機能の追加（記録のバックアップ）

### 長期目標

- 複数デバイス間での同期
- 記録の共有機能（オプション）
- AIによる記録の自動分類・タグ付け
- 記録に基づくレコメンデーション機能

## 成功の指標

1. **ユーザーエンゲージメント**
   - 週次アクティブユーザー数
   - 1ユーザーあたりの記録作成数
   - 記録の閲覧回数

2. **ユーザー満足度**
   - アプリストアでの評価
   - ユーザーフィードバック
   - 継続利用率

3. **技術的指標**
   - アプリの起動速度
   - APIレスポンス時間
   - エラー発生率
