# ユーザーストーリー

## 認証関連

### US-001: ユーザー登録

**ユーザーストーリー**:
> 新規ユーザーとして、メールアドレスとパスワードでアカウントを作成したい。そうすることで、自分の記録を安全に管理できる。

**受け入れ条件**:
- [ ] メールアドレス、パスワード、ユーザー名（必須、25文字以内）を入力して登録できる
- [ ] メールアドレスが重複している場合、エラーメッセージが表示される
- [ ] メールアドレスまたはパスワードが未入力の場合、エラーメッセージが表示される
- [ ] 登録成功後、自動的にログインされる
- [ ] 登録成功後、メイン画面（ギャラリー）に遷移する

**優先度**: 高

---

### US-002: ログイン

**ユーザーストーリー**:
> 登録済みユーザーとして、メールアドレスとパスワードでログインしたい。そうすることで、自分の記録にアクセスできる。

**受け入れ条件**:
- [ ] メールアドレスとパスワードを入力してログインできる
- [ ] メールアドレスまたはパスワードが間違っている場合、エラーメッセージが表示される
- [ ] ログイン成功後、メイン画面（ギャラリー）に遷移する
- [ ] ログイン成功後、JWTトークンがSecureStoreに保存される

**優先度**: 高

---

### US-003: ログアウト

**ユーザーストーリー**:
> ログイン中のユーザーとして、ログアウトしたい。そうすることで、他の人にアカウントを利用されないようにできる。

**受け入れ条件**:
- [ ] 設定画面からログアウトできる
- [ ] ログアウト後、ログイン画面に遷移する
- [ ] ログアウト後、保存されたトークンが削除される
- [ ] ログアウト後、認証が必要なAPIにアクセスできない

**優先度**: 高

---

### US-004: 認証状態の保持

**ユーザーストーリー**:
> アプリを再起動したユーザーとして、再度ログインしなくても自分の記録にアクセスしたい。そうすることで、毎回ログインする手間を省ける。

**受け入れ条件**:
- [ ] アプリ再起動後、保存されたトークンが自動的に読み込まれる
- [ ] トークンが有効な場合、自動的にログイン状態が復元される
- [ ] トークンが無効な場合、ログイン画面に遷移する
- [ ] トークン読み込み中はローディング画面が表示される

**優先度**: 高

---

## 記録管理関連

### US-005: 記録の作成

**ユーザーストーリー**:
> ユーザーとして、写真とテキストで記録を作成したい。そうすることで、日々の体験を簡単に記録できる。

**受け入れ条件**:
- [ ] 写真を選択できる（カメラロールから選択）
- [ ] タイトルを入力できる（任意）
- [ ] 説明を入力できる（任意）
- [ ] 日付を選択できる（必須）
- [ ] カテゴリーを選択できる（任意）
- [ ] タイムライン表示の有無を設定できる（デフォルト: 表示）
- [ ] 記録を作成できる
- [ ] タイトル未入力時は「無題の記録」がデフォルト値になる
- [ ] 作成成功後、ギャラリー画面に戻る
- [ ] 作成した記録がギャラリーに表示される

**優先度**: 高

---

### US-006: 記録の一覧表示

**ユーザーストーリー**:
> ユーザーとして、自分の記録を一覧で見たい。そうすることで、過去の記録を振り返れる。

**受け入れ条件**:
- [ ] 自分の記録が一覧で表示される（表示形式: グリッド/リスト/ブックリスト/タイル）
- [ ] 並び順を切り替えられる（記録日/作成日）
- [ ] カテゴリーでフィルターできる（横スワイプで切り替え）
- [ ] 各記録には写真、タイトル、日付が表示される
- [ ] 記録をタップすると詳細画面に遷移する
- [ ] 削除済みの記録は表示されない
- [ ] 記録がない場合、適切なメッセージが表示される

**優先度**: 高

---

### US-007: 記録の詳細表示

**ユーザーストーリー**:
> ユーザーとして、特定の記録の詳細を見たい。そうすることで、その時の詳細な情報を確認できる。

**受け入れ条件**:
- [ ] 記録の詳細情報（写真、タイトル、説明、日付、カテゴリー）が表示される
- [ ] 記録を編集できるボタンが表示される
- [ ] 記録を削除できるボタンが表示される
- [ ] スワイプで前後の記録に移動できる
- [ ] 画像をズーム・位置調整できる

**優先度**: 高

---

### US-008: 記録の更新

**ユーザーストーリー**:
> ユーザーとして、作成した記録を編集したい。そうすることで、間違いを修正したり、後から情報を追加できる。

**受け入れ条件**:
- [ ] 記録のタイトル、説明、日付、カテゴリーを編集できる
- [ ] 画像を変更できる
- [ ] タイムライン表示の有無を変更できる
- [ ] 更新内容を保存できる
- [ ] 更新成功後、詳細画面に反映される
- [ ] 自分の記録のみ編集可能

**優先度**: 高

---

### US-009: 記録の削除

**ユーザーストーリー**:
> ユーザーとして、不要になった記録を削除したい。そうすることで、記録を整理できる。

**受け入れ条件**:
- [ ] 記録を削除できる
- [ ] 削除前に確認ダイアログが表示される
- [ ] 削除後、ギャラリーから記録が消える
- [ ] 削除は論理削除（データベースには残るが表示されない）
- [ ] 自分の記録のみ削除可能

**優先度**: 高

---

### US-010: カテゴリーフィルター

**ユーザーストーリー**:
> ユーザーとして、特定のカテゴリーの記録だけを見たい。そうすることで、テーマごとに記録を振り返れる。

**受け入れ条件**:
- [ ] カテゴリーでフィルターできる
- [ ] フィルター適用後、該当カテゴリーの記録のみ表示される
- [ ] フィルターを解除できる
- [ ] フィルター状態が視覚的に分かる

**優先度**: 中

---

## カテゴリー管理関連

### US-011: カテゴリーの作成

**ユーザーストーリー**:
> ユーザーとして、記録を分類するためのカテゴリーを作成したい。そうすることで、記録を整理できる。

**受け入れ条件**:
- [ ] カテゴリー名を入力してカテゴリーを作成できる
- [ ] カテゴリー名は必須
- [ ] 作成したカテゴリーがカテゴリー一覧に表示される
- [ ] 作成したカテゴリーを記録作成時に選択できる

**優先度**: 中

---

### US-012: カテゴリーの一覧表示

**ユーザーストーリー**:
> ユーザーとして、自分のカテゴリー一覧を見たい。そうすることで、どのカテゴリーがあるか確認できる。

**受け入れ条件**:
- [ ] 自分のカテゴリーが一覧で表示される
- [ ] 各カテゴリーには名前が表示される
- [ ] カテゴリーを編集できる（長押しで編集モーダル）
- [ ] カテゴリーを削除できる

**優先度**: 中

---

### US-013: カテゴリーの更新

**ユーザーストーリー**:
> ユーザーとして、作成したカテゴリーを編集したい。そうすることで、カテゴリー名を変更できる。

**受け入れ条件**:
- [ ] カテゴリー名を編集できる
- [ ] 更新内容を保存できる
- [ ] 更新後、カテゴリー一覧に反映される
- [ ] 更新後、記録作成時のカテゴリー選択にも反映される

**優先度**: 中

---

### US-014: カテゴリーの削除

**ユーザーストーリー**:
> ユーザーとして、不要になったカテゴリーを削除したい。そうすることで、カテゴリーを整理できる。

**受け入れ条件**:
- [ ] カテゴリーを削除できる
- [ ] 削除前に確認ダイアログが表示される
- [ ] 削除後、カテゴリー一覧から消える
- [ ] 削除されたカテゴリーに紐づく記録のcategory_idはNULLになる

**優先度**: 中

---

## プロフィール管理関連

### US-015: プロフィール情報の表示

**ユーザーストーリー**:
> ユーザーとして、自分のプロフィール情報を確認したい。そうすることで、現在の設定を確認できる。

**受け入れ条件**:
- [ ] ユーザー名、メールアドレス、アバター画像が表示される
- [ ] 設定画面からプロフィール情報を確認できる

**優先度**: 中

---

### US-016: プロフィール情報の更新

**ユーザーストーリー**:
> ユーザーとして、自分のユーザー名やアバター画像を変更したい。そうすることで、プロフィールを更新できる。

**受け入れ条件**:
- [ ] ユーザー名を変更できる（25文字以内）
- [ ] 自己紹介を変更できる
- [ ] 公開設定（public/private）を変更できる
- [ ] アバター画像を変更できる
- [ ] 更新内容を保存できる
- [ ] 更新後、プロフィール画面に反映される
- [ ] メールアドレスは変更できない

**優先度**: 中

---

## 設定機能関連

### US-017: 表示設定

**ユーザーストーリー**:
> ユーザーとして、記録一覧の表示形式や並び順を設定したい。そうすることで、自分の好みに合わせて一覧を見られる。

**受け入れ条件**:
- [ ] 表示形式（グリッド/リスト/ブックリスト/タイル）を選択できる
- [ ] 並び順（記録日/作成日）を選択できる
- [ ] 設定変更後、一覧画面に即座に反映される
- [ ] 設定はアプリ再起動後も保持される

**優先度**: 低

---

### US-018: 言語設定

**ユーザーストーリー**:
> ユーザーとして、アプリの表示言語を変更したい。そうすることで、自分の言語でアプリを使える。

**受け入れ条件**:
- [ ] 日本語と英語を選択できる
- [ ] 設定変更後、アプリ全体の言語が即座に変更される
- [ ] 設定はアプリ再起動後も保持される

**優先度**: 低

---

## インサイト機能関連

### US-019: 統計情報の表示

**ユーザーストーリー**:
> ユーザーとして、自分の記録の統計情報を見たい。そうすることで、記録の傾向を把握できる。

**受け入れ条件**:
- [ ] 総記録数が表示される
- [ ] 今月の記録数が表示される
- [ ] 今週の記録数が表示される
- [ ] カテゴリー数が表示される
- [ ] 統計情報は自分の記録のみを集計する

**優先度**: 低

---

## 画像管理関連

### US-020: 画像のアップロード

**ユーザーストーリー**:
> ユーザーとして、記録やカテゴリーに画像をアップロードしたい。そうすることで、視覚的に記録を豊かにできる。

**受け入れ条件**:
- [ ] カメラロールから画像を選択できる
- [ ] 画像をアップロードできる
- [ ] アップロード後、画像が表示される
- [ ] 画像サイズ制限（カテゴリー画像は5MB以下）が適用される

**優先度**: 高

---

### US-021: タイムライン表示設定

**ユーザーストーリー**:
> ユーザーとして、記録をスレッドのタイムラインに表示するか設定したい。そうすることで、フォロワーに共有するかどうかを選べる。

**受け入れ条件**:
- [ ] 記録作成・編集時にタイムライン表示の有無を設定できる
- [ ] デフォルトは表示（ON）
- [ ] 設定内容が保存される

**優先度**: 中

---

### US-022: ユーザー検索

**ユーザーストーリー**:
> ユーザーとして、他のユーザーを検索したい。そうすることで、フォローしたい人を見つけられる。

**受け入れ条件**:
- [ ] 検索文字列を入力してユーザーを検索できる
- [ ] 公開ユーザーは名前で部分一致検索できる
- [ ] 非公開ユーザーは検索キー完全一致時のみ検索できる
- [ ] 検索結果からフォローできる

**優先度**: 中

---

### US-023: フォロー

**ユーザーストーリー**:
> ユーザーとして、気になるユーザーをフォローしたい。そうすることで、その人の記録をタイムラインで見られる。

**受け入れ条件**:
- [ ] 他ユーザーをフォローできる
- [ ] フォロー解除できる
- [ ] フォロー中・フォロワー一覧を表示できる
- [ ] 自分自身はフォローできない

**優先度**: 中

---

### US-024: タイムライン表示

**ユーザーストーリー**:
> ユーザーとして、フォロー中の人の記録をタイムラインで見たい。そうすることで、友達の近況を確認できる。

**受け入れ条件**:
- [ ] フォロー中ユーザーの直近7日間の記録が表示される
- [ ] タイムライン表示がONの記録のみ表示される
- [ ] 記録から投稿者のプロフィールに遷移できる
- [ ] フォローしていない場合は空のタイムラインが表示される

**優先度**: 中

---

### US-025: 他ユーザープロフィール表示

**ユーザーストーリー**:
> ユーザーとして、他ユーザーのプロフィールと投稿を見たい。そうすることで、フォローするかどうか判断できる。

**受け入れ条件**:
- [ ] 他ユーザーの名前、自己紹介、アバターが表示される
- [ ] 他ユーザーの投稿一覧が表示される
- [ ] フォロー/フォロー解除ボタンが表示される
- [ ] 記録をタップして詳細を表示できる

**優先度**: 中

---

## 優先度の定義

- **高**: コア機能、アプリの基本動作に必須
- **中**: 重要な機能だが、なくても基本動作は可能
- **低**: 付加価値機能、ユーザビリティ向上
